# 과제 제안서

## [과제 기본 정보]

**과제명:**
- 국문: 생성형 AI 기반 그린수소 프로젝트 금융조달성(Bankability) 분석 에이전트 개발
- 영문: Development of Generative AI-based Bankability Analysis Agent for Green Hydrogen Projects

**과제 요약:**
본 과제는 그린수소 생산 프로젝트의 금융조달 가능성(Bankability)을 자동 평가하고 개선 방안을 도출하는 AI 에이전트 시스템을 개발한다. 수전해 설비의 기술적 파라미터(용량, 효율, CAPEX 등)와 금융 조건(부채비율, 금리, 대출기간), 그리고 **정부 인센티브(세액공제, 보조금, 탄소배출권 등)**를 입력받아 몬테카를로 시뮬레이션 기반 확률적 현금흐름을 생성하고, Claude API와 Tool Use를 활용하여 DSCR(원리금상환비율), LLCR/PLCR(대출생애/프로젝트생애 커버리지), Covenant 준수 여지, 스트레스 테스트 등 프로젝트 파이낸스 핵심 지표를 자동 분석한다. 금융기관 심사 관점의 리스크 요인을 식별하고, 최적 레버리지 구조, 적립금 요구사항, Offtake 계약 구조, **인센티브 활용 최적화** 등 구체적인 Bankability 개선 권고안을 대화형 인터페이스로 제공함으로써, 그린수소 프로젝트의 금융 접근성을 높이고 탄소중립 인프라 투자 확대에 기여한다.

**분야:** 에너지 금융 최적화 / 그린수소 프로젝트 파이낸스

**적용 AI 기술:**
- 생성형 AI (Large Language Model - Claude)
- Tool Use / Function Calling 기반 자율 계산 에이전트
- Prompt Caching을 통한 도메인 지식 최적화
- 확률적 시뮬레이션 (Monte Carlo)

## [1. 기술개발의 필요성]

### 현황 및 문제점

**현재 탄소 중립 관리 방식의 한계점:**
- 그린수소는 탄소중립 달성의 핵심 수단이나, 높은 초기 투자비용과 불확실한 수익성으로 인해 프로젝트 파이낸싱 확보가 최대 난제
- 국내 수전해 프로젝트 대부분이 실증 단계에 머물러 있으며, 상업적 규모의 프로젝트는 금융조달 실패로 지연되는 사례 다수
- 기존 재무 분석은 단순 DCF(현금흐름할인) 모델에 의존, 프로젝트 파이낸스 특유의 Covenant, DSCR Tail, Reserve 요구사항 등을 반영하지 못함

**향후 규제 강화에 따른 예상 문제:**
- 2030년 NDC 목표 달성을 위해 청정수소 의무 사용 비율 확대 예정 (2030년 수소 공급량 390만톤 중 청정수소 목표)
- EU CBAM, 미국 IRA 등 국제 규제로 그린수소 인증 기준 강화
- 금융기관의 ESG 심사 강화로 Bankability 미충족 프로젝트는 금융시장 접근 자체가 불가

**다양한 정부 인센티브 활용의 복잡성:**
- 투자세액공제(ITC), 생산세액공제(PTC), CAPEX 보조금, 운영 보조금 등 다양한 인센티브 존재
- 인센티브별 적용 조건, 기간, 중복 적용 가능 여부 등 복잡한 규정
- 인센티브가 현금흐름과 Bankability에 미치는 영향을 정량적으로 분석하기 어려움
- 탄소배출권, 청정수소 인증 프리미엄 등 추가 수익원의 불확실성

### 기존 기술의 한계

**기존 통계/수동 관리 모델의 기술적 난제:**
- 엑셀 기반 재무모델은 수백 개의 변수 간 상호작용을 반영하기 어려움
- DSCR 스트레스 테스트, 민감도 분석 등은 수작업으로 수일 소요
- 금융기관별 상이한 Covenant 조건을 즉시 반영한 시나리오 분석 불가
- 비재무 전문가(엔지니어, 사업개발자)가 금융 구조 최적화를 직접 수행하기 어려움

### AI 기술 적용의 필연성

**복잡한 비정형 데이터 처리 및 실시간 자율 제어의 필요성:**
- 프로젝트 파이낸스 심사는 정량적 지표(DSCR, IRR)와 정성적 판단(시장 리스크, Offtake 신뢰도)의 복합적 평가 필요
- 생성형 AI의 Tool Use 기능을 통해 복잡한 금융 계산을 자율 수행하고, 자연어로 인사이트 제공 가능
- 대화형 인터페이스로 "What-if" 시나리오 탐색 및 즉각적 피드백 제공

### 국내외 기술 수준 및 시장 동향

**글로벌 선진 기술과의 격차:**
- 해외: Bloomberg NEF, Wood Mackenzie 등이 수소 프로젝트 금융 분석 도구 제공하나, 고가의 구독료(연간 수천만원)와 범용적 모델로 국내 금융 환경 미반영
- 국내: 수소 특화 금융 분석 도구 부재, 대부분 회계법인/금융자문사의 수작업 모델에 의존

**타겟 시장 규모:**
- 글로벌 그린수소 시장: 2023년 38억 달러 → 2030년 720억 달러 (CAGR 52.2%, Allied Market Research)
- 국내 수소경제 투자: 2030년까지 43조원 투자 계획 (정부 수소경제 로드맵)
- 프로젝트 파이낸스 시장: 국내 재생에너지 PF 시장 연간 10조원 이상, 수소 분야 급성장 예상

## [3. 아이디어 요약 및 에이전트(Agent) 정의]

### 아이디어 요약
> "AI가 수소 프로젝트의 금융조달 장벽을 분석하고 해결책을 제시한다"

### 에이전트 제안 배경

**현장 데이터 분석을 통한 문제 정의:**
- 국내 수전해 프로젝트 10건 검토 결과, 70%가 DSCR 1.3x 미달로 금융조달 난항
- 주요 원인: 전력비용 변동성, Offtake 계약 미확보, 과도한 레버리지 설계
- 프로젝트 개발사의 금융 구조 최적화 역량 부족이 핵심 병목

**도출된 인사이트:**
- 기술적 타당성이 확보된 프로젝트도 금융 구조 설계 미흡으로 좌초
- 금융기관 관점의 "심사 통과 조건"을 사전에 파악하면 성공률 대폭 향상 가능

### 핵심 가치 (Core Value)

| 가치 | 설명 |
|------|------|
| **Bankability 자가진단** | 프로젝트 금융조달 가능성 즉시 확인 및 개선점 도출 |
| **구조 최적화 가이드** | 최적 부채비율, 적립금, Covenant 수준 자동 제안 |
| **리스크 사전 식별** | 금융기관이 지적할 리스크 요인 선제적 파악 및 완화 방안 제시 |
| **인센티브 활용 최적화** | 세액공제, 보조금, 탄소배출권 등 정부 지원 정책의 최적 조합 분석 |
| **대화형 금융자문** | 비전문가도 자연어 질문으로 복잡한 금융 분석 수행 |
| **다각적 분석 역량** | 수백 개 시나리오를 동시 탐색하여 최적 조건 발견 (Grid Search, AI 최적화) |
| **SaaS 수익화 모델** | 분석 건수 기반 과금으로 재무 자문 서비스의 확장 가능한 수익 모델 구축 |

### 도입 기대효과

### 데이터 및 AI 모델 연계

**데이터 흐름:**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   수집      │     │   전처리    │     │   분석      │     │   피드백    │
│             │     │             │     │             │     │             │
│ • 설비 스펙 │────▶│ • 단위 변환 │────▶│ • Monte    │────▶│ • 결과 저장 │
│ • 금융 조건 │     │ • 검증      │     │   Carlo    │     │ • 시나리오  │
│ • 시장 가정 │     │ • 정규화    │     │ • AI 분석  │     │   비교      │
│             │     │             │     │ • Tool Use │     │ • 학습 개선 │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

**AI 모델 연계 방식:**

1. **Structured Context Injection**
   - 시뮬레이션 결과를 구조화된 JSON으로 변환
   - 입력 요약, KPI 요약, 금융 구조, 민감도, 리스크 폭포수 포함

2. **Tool Use (Function Calling)**
   - 8개 금융 계산 도구 정의:
     - `calculate_dscr_stress_test`: DSCR 스트레스 테스트
     - `calculate_optimal_leverage`: 최적 레버리지 계산
     - `calculate_debt_sizing`: 부채 규모 산정
     - `calculate_reserve_requirements`: 적립금 요구사항
     - `analyze_covenant_headroom`: Covenant 여유도
     - `calculate_llcr_plcr`: LLCR/PLCR 산출
     - `calculate_breakeven_price`: 손익분기 가격
     - `assess_offtake_structure`: Offtake 구조 평가

3. **Prompt Caching**
   - 도메인 지식(프로젝트 파이낸스, 수전해 기술) 시스템 프롬프트 캐싱
   - API 비용 절감 및 응답 속도 향상

4. **인센티브 시뮬레이션 연계**
   - 세액공제/보조금이 실질 CAPEX 및 현금흐름에 미치는 영향 자동 계산
   - 기간 제한 인센티브(PTC, 운영보조금)의 종료 후 현금흐름 변화 반영
   - AI가 인센티브 활용 최적화 전략 및 정책 리스크 분석 제공

**다각적 분석의 가치:**

기존 방식에서는 시간과 비용 제약으로 2-3개 시나리오만 검토 가능했으나, 본 솔루션을 통해:

1. **수백 개 변수 조합 탐색** - Grid Search로 최적 설비용량/전력가격/부채비율 조합 발견
2. **다양한 리스크 시나리오** - 전력가격 변동, 수소가격 변동, 이자율 변동 등 복합 시나리오 분석
3. **인센티브 조합 최적화** - ITC, PTC, 보조금의 다양한 조합별 효과 비교
4. **AI 기반 목표 지향 탐색** - "NPV 100억 이상, DSCR 1.3 이상" 등 목표 조건 달성 경로 자동 탐색

이를 통해 재무 자문 회사는 더 높은 품질의 분석을 제공하면서도 분석 건수에 따른 과금으로 수익을 창출할 수 있으며, 고객은 더 다양한 관점에서의 분석을 통해 의사결정 품질을 높일 수 있습니다.

### 인센티브의 Bankability 영향

1. **CAPEX 감소 효과**
   - ITC + 보조금 → 실질 투자금액 감소
   - 부채 규모 감소 → 원리금 상환 부담 완화 → DSCR 개선

2. **운영 현금흐름 증가**
   - PTC, 운영보조금, 탄소배출권, 청정수소 프리미엄
   - 적용 기간 내 현금흐름 안정화 → DSCR 직접 개선

3. **대출기관 관점 고려사항**
   - 인센티브의 법적 확정성 및 정책 지속성
   - 기간 제한 인센티브 종료 후 현금흐름 변화
   - 인센티브 의존도가 높은 구조의 리스크
